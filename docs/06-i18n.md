# 06 — Internationalization (i18n)

Guildora apps support multiple languages. The host selects the translation file based on the user's locale.

## File Structure

```
src/i18n/
├── en.json   ← English (required, fallback)
└── de.json   ← German (required for Guildora default locales)
```

Both files must have **identical keys**.

## Translation Files

```json
// src/i18n/en.json
{
  "title": "Community Points",
  "points": "Points",
  "leaderboard": "Leaderboard",
  "rank": "Rank",
  "member": "Member",
  "yourPoints": "Your points",
  "noData": "No data yet.",
  "awardPoints": "Award Points",
  "awardSuccess": "Awarded {amount} points to {username}.",
  "error": {
    "notFound": "Not found.",
    "forbidden": "You don't have permission to do this."
  }
}
```

```json
// src/i18n/de.json
{
  "title": "Community-Punkte",
  "points": "Punkte",
  "leaderboard": "Bestenliste",
  "rank": "Rang",
  "member": "Mitglied",
  "yourPoints": "Deine Punkte",
  "noData": "Noch keine Daten.",
  "awardPoints": "Punkte vergeben",
  "awardSuccess": "{amount} Punkte an {username} vergeben.",
  "error": {
    "notFound": "Nicht gefunden.",
    "forbidden": "Du hast keine Berechtigung dafür."
  }
}
```

## Usage in Vue

```vue
<template>
  <div>
    <h1>{{ t('title') }}</h1>
    <p>{{ t('yourPoints') }}: {{ points }}</p>

    <!-- Nested key -->
    <p class="error">{{ t('error.notFound') }}</p>

    <!-- Interpolation -->
    <p>{{ t('awardSuccess', { amount: 10, username: 'Alice' }) }}</p>
  </div>
</template>

<script setup lang="ts">
const { t, locale } = useI18n()
// locale.value → 'en' | 'de' | ...
</script>
```

## Pluralization

Use array syntax for plural forms:

```json
{
  "pointCount": ["1 point", "{count} points"]
}
```

```typescript
t('pointCount', { count: 1 })   // "1 point"
t('pointCount', { count: 42 })  // "42 points"
```

## Tips

- Always define `en.json` first, then translate
- Keep keys short but descriptive (`awardSuccess` not `theMessageShownWhenPointsAreAwarded`)
- Use nested objects for grouping (`error.notFound`, `error.forbidden`)
- All user-visible strings should go through `t()` — never hardcode text in templates
